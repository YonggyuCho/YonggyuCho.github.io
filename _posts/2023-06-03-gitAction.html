--- layout: post title: "gitAction 공부." subtitle: "CI/CD ." date: 2023-06-03 background: '' --- CI/CD란 CI (Continuous Integration, 지속통합) CD (Contiuous Deployment, 지속배포)

CI와 CD를 합쳐 CI/CD로 많이부른다

어플리케이션 개발단계부터 배포까지 자동화를통해

빠르고 효율적이게 사용자에게 배포하는 개발프로세스.

CI단계에서는 Main Respository에

코드 변경사항을 주기적으로 빈번하게 Merge해야한다.

주기적인 단위는 아주 작은 단위여야한다.

지속적인 Merge가 자동으로 Build되고 UnitTest까지 자동화가 된다.

실패했을경우 Red, 성공했을경우 Green으로 알려준다.

이 CI를 통해 개발 생산성이 향상되고

버그수정이 용이하고

문제점을 빠르게 발견이 가능하다.

*코드의 퀄리티가 향상

CD 릴리즈와 배포단계이다.

결국 CI/CD는

Build, Test, 릴리즈, 배포 단계들이 전부 자동화이다

여러 tool이 존재하지만 그중하나 Github Actions를 알아보자

Github Actions GithubAction은 5가지개념이 중요하다 Events, Workflows, Jobs, Actions, Runners Events

on : push

github에서 발생하는 특정한 이벤트(사건)들을 지정한다.

예) main Branch로 merge하는과정, 커밋을 푸쉬하는과정, 이슈를 누군가 여는과정 등

Workflows

특정한 이벤트가발생했을때 내가 어떤일을 수행하고싶냐를 지정하는게 workflows

자동화 하고자하는것을 workflow안에 지정할 수있다.

이 Workfows 안에는 여러개의 Job이 존재

Jobs

하나의 Job단위로 특정 행동을 한다

예) 어떤Job은 유닛테스트를하고 또다른Job은 E2E테스트를 한다.

여러개의 Job들은 동시다발적으로 실행된다.하지만 Job끼리의 순서 지정도 가능

각각 Job안에는 Step(순서)을 지정할수있다

예) Step1: action Checkout , Step2 : npm test

Actions

Step을 명시하는 과정에서 Actions를 사용한다.

git에서 제공하는 여러개의 유용한 Actions이다.

Runners

Job들을 실행하는것이 Runners
