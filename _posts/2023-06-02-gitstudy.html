Git 정리

git init (파일이름) : 저장소 생성하기

- 디렉토리 내부에 .git 디렉토리가 생긴다

git status : commit 상태확인

git add (파일이름) : 파일의 상태를 commit으로

-add를 하는이유는 파일을 select해서 commit 하기위해

git commit : commit 상태인 파일들을 전부 저장소로 보낸다

git commit -m : editor(vim) 이 안뜨도록 저장소로 보낸다

git log : commit한 history(hash ID)가 나온다.

git log -p : 변경한 내용, 좀더 상세하게 보여준다.

git log --all : 모든branch의 로그를 보여준다

git log --all --graph --decorate : 모든branch와 연결된 정보들을 보여준다.

git diff (hash ID)..(hash ID) : 소스사이에 변화를 보여준다.

git diff : ID를 지정하지않아도 staged한 파일의 변화를 보여준다

git restore (파일이름) : 기존 변화 정보 제거햣

git reset (hash ID) : commit한 특정 시점으로 돌아가기

- reset은 안하는게 좋다(협업시 충돌이 일어난다)

Branch

Branch는 여러개의 작업을 가능케한다.

HEAD : 현재의 branch

git branch : 현재 branch목록을보여준다

git branch (이름) : branch 생성

git checkout (branch 이름) : HEAD를 변경한다.

git merge (branch 이름) : branch에서 변경된내용을 HEAD로 적용

git revert : 일부 기존 commit 되돌리기

commit 되지않은것은 branch로 옮겨도 같이 따라온다

그래서 사용하는것이 stash

git stash : 숨긴파일느낌 add한 파일을 stash list로 옮긴다

git stash drop (파일) : stash파일 없애기

git stash apply (파일) : stash파일 뺴오기

tag 소스의 특정시점을 tag로 지정

github

public private 차이

public은 내코드를 공유는 가능하지만 수정은 불가

private는 공유도 X

git remote add (origin) (주소)

(origin)은 이름이다 원격저장소를 짧게 요약

github에 올리는순서

git init

git add (파일)

git commit -m "메세지"

git branch -m (main)

git remote add (origin) (원격주소)

git push -u origin main
